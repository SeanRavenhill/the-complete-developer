# The Complete Developer

## Master the Full Stack with Typescript, React, Next.js, MongoDB, and Docker

### Written by: Martin Krause | Published by: No Starch Press

This repo will provide my version control and history as I work through this
book in the pursuit of learning and comprehending the process, systems, architecture
and technology required in developing a full stack application.

The Frontend, The Middleware and The Backend.

This book covers the **_M.E.R.N_** _MongoDB (database), Express.js (backend framework), React.js (frontend library), and Node.js (runtime environment)_ Tech Stack. With the caveat that JavaScript and React.js will be expanded upon to make use of TypeScript and Next.js

It is March 17th 2026 as I start this. A time of massive AI hype/noise (or maybe it's not and this exercise is futile in the end), but I got into coding and programming as I might be one of the few people left whom actually enjoyed the process of learning, of using my own grey matter between my skull to abstract problems into working solution via code.

It get's up my nose to read so many blogs and posts where "coding was never the skill". If that is the case, then I don't suppose a mathematician mastering basics like arthematic was ever the skill either.

I want to understand the **_why_** and the **_how_**, and I want to be able to do that without outsourcing my thinking to an **_LLM_** model or the latest new and shiny **_AI Assisted IDE_** or **_Agent_**.

Since I shipped my first website builds, which I felt were a type of \*_*software*_, albeit basic examples, this was a craft and skill that I felt resonated with me. I enjoyed it. I enjoyed the process.

So maybe this is the new **_"Caveman Coding"_** and this exercise is indeed a futile one. I will have at least exercised my brain, read a book, and build something while learning some new skills. Even if I am 10 years or so too late to this party.
